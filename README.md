# AutoBattlerGame1.1

## Описание проекта
Прототип минималистичного 2D автобаттлера с элементами выживания, разработанный в рамках учебной практики.Основной фокус проекта — архитектура управления данными (Data-driven design) и глубокая оптимизация памяти. 

## Правила игры
Игрок оказывается на арене, где на него непрерывно наступают волны противников. 
* **Цель:** Продержаться как можно дольше, маневрируя и уничтожая врагов.
* **Бой:** Атака происходит автоматически, когда враг попадает в радиус поражения.
* **Прогрессия:** Из поверженных противников выпадают кристаллы опыта. При повышении уровня игра ставится на паузу, и игрок может выбрать один из перков (повышение урона, скорости или лечение).
* **Поражение:** Игра заканчивается, если полоска здоровья игрока опускается до нуля.

## Управление
* **W, A, S, D / Стрелки** — Перемещение персонажа.
* **Левая кнопка мыши (ЛКМ)** — Взаимодействие с интерфейсом (выбор улучшений).
* **Escape (Esc)** — Пауза / Вызов внутриигрового меню.

## Технические особенности реализации
* **Data-Driven Design:** В проекте реализован полный отказ от хардкода. Все базовые характеристики врагов, параметры оружия и свойства апгрейдов хранятся в `ScriptableObject`.
* **Оптимизация памяти (Object Pool):** Выполнено жесткое требование по оптимизации.Реализован паттерн Object Pool для спавна врагов, снарядов и частиц опыта. Во время геймплея исключены вызовы `Instantiate` и `Destroy`, что гарантирует стабильный FPS без спайков Garbage Collector даже при наличии 100+ активных сущностей на сцене.
* **UI Архитектура:** Разделение логики интерфейса и игровых систем с использованием событийно-ориентированного подхода.

## Запуск проекта
1. Скачайте архив с релизным билдом.
2. Распакуйте архив в удобную папку.
3. Запустите исполняемый `.exe` файл.
4. Проект является полностью автономным и не требует установки среды Unity.
