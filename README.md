# AutoBattlerGame1.1

## Описание проекта
[cite_start]Прототип минималистичного 2D автобаттлера с элементами выживания, разработанный в рамках учебной практики[cite: 17]. [cite_start]Основной фокус проекта — архитектура управления данными (Data-driven design) и глубокая оптимизация памяти[cite: 19]. 

## Правила игры
[cite_start]Игрок оказывается на арене, где на него непрерывно наступают волны противников[cite: 18]. 
* [cite_start]**Цель:** Продержаться как можно дольше, маневрируя и уничтожая врагов[cite: 18].
* [cite_start]**Бой:** Атака происходит автоматически, когда враг попадает в радиус поражения[cite: 18].
* [cite_start]**Прогрессия:** Из поверженных противников выпадают кристаллы опыта[cite: 18]. [cite_start]При повышении уровня игра ставится на паузу, и игрок может выбрать один из перков (повышение урона, скорости или лечение)[cite: 18].
* **Поражение:** Игра заканчивается, если полоска здоровья игрока опускается до нуля.

## Управление
* **W, A, S, D / Стрелки** — Перемещение персонажа.
* **Левая кнопка мыши (ЛКМ)** — Взаимодействие с интерфейсом (выбор улучшений).
* **Escape (Esc)** — Пауза / Вызов внутриигрового меню.

## Технические особенности реализации
* [cite_start]**Data-Driven Design:** В проекте реализован полный отказ от хардкода[cite: 20]. [cite_start]Все базовые характеристики врагов, параметры оружия и свойства апгрейдов хранятся в `ScriptableObject`[cite: 20].
* [cite_start]**Оптимизация памяти (Object Pool):** Выполнено жесткое требование по оптимизации[cite: 21]. [cite_start]Реализован паттерн Object Pool для спавна врагов, снарядов и частиц опыта[cite: 21]. [cite_start]Во время геймплея исключены вызовы `Instantiate` и `Destroy`, что гарантирует стабильный FPS без спайков Garbage Collector даже при наличии 100+ активных сущностей на сцене[cite: 21, 22].
* **UI Архитектура:** Разделение логики интерфейса и игровых систем с использованием событийно-ориентированного подхода.

## Запуск проекта
1. Скачайте архив с релизным билдом.
2. Распакуйте архив в удобную папку.
3. Запустите исполняемый `.exe` файл.
4. [cite_start]Проект является полностью автономным и не требует установки среды Unity[cite: 87].
